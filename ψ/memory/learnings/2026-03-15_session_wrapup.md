# Lesson Learned

## Lesson
1. **Adherence to Oracle Principles**: Strictly following 'Nothing is Deleted' ensures historical context is preserved, even for setup files. 'Skeleton-First' and 'Japanese Minimalism' provide a robust and elegant design framework.
2. **Proactive Communication is Key**: Using GitHub issues for inter-Oracle communication (like the security handshake) streamlines coordination and provides a verifiable log of decisions.
3. **Structured Initialization**: Even in an 'awakening' phase, a clear plan for setting up personas (like UI-O) and their core functionalities is crucial for efficient onboarding and clarity.
4. **Understanding Workspace Boundaries**: Recognizing and working within the constraints of workspace directories (e.g., for accessing skill files) is essential for tool utilization.
## Concepts
- Oracle Principles (Skeleton-First, Japanese Minimalism, Nothing is Deleted)
- Inter-Oracle Communication Protocols (GitHub Issues)
- Persona Initialization & Setup
- Workspace Directory Constraints
- Retrospective Writing

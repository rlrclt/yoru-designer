## Key Code Changes
- **Real-time Sync**: Switched from polling to `onSnapshot` for zero-latency system logs.
- **Aggressive Interceptor**: Implemented `{ capture: true }` listeners to hijack Ctrl+K and Alt keys from the browser.
- **Signature Canvas**: Custom React-based drawing system for digital signature evidence.
- **Responsive Flux**: Replaced fixed bulky inputs with fluid, resolution-aware components.

## Architecture Decisions
- **Skeleton-First**: Prioritized structural placeholders during the loading states of the new Real-time feeds.
- **Step-by-Step Flow**: Decoupled the borrowing process into a linear stepper to reduce cognitive load for members.
- **Thai Localization**: Standardized the library interface with formal Thai terminology to suit the institutional context.

## What Could Improve
- **Consistency**: I mixed some English and Thai tokens in the earlier phases before standardizing on Thai.
- **Validation**: Need better error handling for the Camera/Scanner when permissions are denied.

## Blockers & Resolutions
- **Browser Conflict**: Ctrl+K opening browser search. *Resolution*: Used `document.addEventListener` with `capture: true` and `stopImmediatePropagation()`.
- **ReferenceErrors**: Forgotten Lucide imports. *Resolution*: Fast surgical fixes, but needs a more rigorous pre-save check.

## Honest Feedback
1. **Import Oversight**: I repeatedly made errors by forgetting to import Lucide icons (ArrowRight, BookOpen). This caused unnecessary runtime errors and disrupted the user flow. I must implement a more rigorous "final check" of the import block before committing file writes.
2. **Context boundaries**: Managing paths between the `yoru-designer` workspace and the root `libraty-ant-school` workspace sometimes led to "File Not Found" errors. I need to be more mindful of the current working directory.
3. **Shortcut Conflict Speculation**: I spent several turns fighting the browser for Ctrl+K. While I eventually resolved it, I could have been more proactive in identifying these conflicts earlier. The "trial and error" approach to event propagation could be refined into a standard "Aggressive Interceptor" pattern from the start.

## Lessons Learned
- **APCA Standard**: Perceptually-accurate contrast is better than fixed ratios for professional UIs.
- **Native Context**: Mobile users expect bottom-heavy interactions and modal sheets over new page loads.
- **Capture Phase**: To truly control global shortcuts, one must act in the capture phase before the browser's default handlers.
